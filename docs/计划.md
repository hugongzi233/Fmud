# 一比一复刻计划：Android MUD 客户端 → Vue.js Web

## 开发规范

- **一次 Agent 运行不要开过长的上下文**。任务要拆分、上下文要精简，避免一次塞太多文件/代码。
- **目标：一比一复刻到 Vue.js + WebSocket**，保持功能 1:1 对齐，不做功能增删。
- **Git 标准开发流程**：所有开发在特性分支进行，禁止直接 commit 到 main，开发完成后通过 Pull Request 合并（参考下方 Git 工作流）。

---

## Git 标准工作流

### 分支命名
```
feature/<功能名>     # 新功能开发
bugfix/<问题名>      # bug 修复
hotfix/<问题名>     # 紧急修复
docs/<文档名>        # 文档更新
refactor/<模块名>   # 重构
```

### 开发流程
```
1. 从 main 新建分支：git checkout -b feature/xxx
2. 开发 + 频繁 commit（每完成一个小功能就 commit）：
   git add <文件>
   git commit -m "feat: 完成 xxx 功能"
3. 阶段开发完成后推送远程：
   git push -u origin feature/xxx
4. 提交 Pull Request，在 description 中说明：
   - 改了什么
   - 为什么改
   - 测试方式（自测步骤）
5. 至少 1 人 review 通过后方可合并到 main
6. 合并后删除分支
```

### Commit 规范
```
feat: 新功能
fix: bug 修复
docs: 文档更新
style: 格式调整（不影响功能）
refactor: 重构
chore: 构建/工具/依赖
test: 测试相关
```

### 注意事项
- **每条 commit 应是原子性的**，一个 commit 只做一件事
- **禁止大规模 commit**，不要把所有文件一次性 commit
- **PR 需要描述清楚改了什么**，方便 reviewer
- **合并后立即删除分支**，保持分支列表清洁
- **冲突第一时间处理**，不要让冲突堆积

## 八、错误记录（避免重犯）

### 错误1：跳过 PR 直接合并到 main
**时间**：第一轮开发（2026-05-02）
**错误行为**：代码在 `feature/web-client-implementation` 分支完成后，直接 `git merge` 到 main，没有创建 PR、没有 review、没有等待。

**正确做法**：必须创建 PR → 至少 1 人 review → 通过后才能合并。

**根因**：当时 `gh` 命令不存在，我图省事跳过了流程。但 git flow 的核心就是不能跳过 review，即使单人开发也要等 self-review 或另一个 agent review 后再合并。

**避免措施**：
- 以后永远不许直接 merge 到 main
- 如果 `gh` 不可用，就在 GitHub 网页上手动创建 PR
- 每次开发前检查当前分支是否基于最新的 origin/main
- 开发前先 `git fetch origin && git log --oneline -5 origin/main` 确认无他人进度冲突

### 错误2：开发前没有比较远程进度
**错误行为**：没有在开发前先 `git fetch` 检查 remote 是否有其他人新的 commit，直接基于旧状态开发，导致可能覆盖他人工作。

**避免措施**：
- 每次开始新任务前，先 `git fetch origin && git log --oneline -5 origin/main` 确认最新状态
- 如果 remote 比本地新，先 `git pull --rebase origin main` 拉取后再开始
- 多人协作场景下，每条特性分支的开发前都要确认基于最新的 main

## 九、开发检查清单（每次开始前必做）

```
1. git fetch origin
2. git log --oneline -3 origin/main  ← 确认无他人新进度
3. git status  ← 确认本地无未 commit 的脏状态
4. 如果要新功能：git checkout -b feature/xxx（从最新的 origin/main 新建）
5. 如果要继续开发：git rebase origin/main（变基到最新）
6. 开发 + commit
7. push 后创建 PR
8. 等 review 通过后再合并
```

## 十、Android ↔ Web 关键映射

| Android 行为 | Web 适配 |
|-------------|---------|
| 进程级退出 `System.exit(0)` | 清理 Pinia store + WebSocket close + router.push('/') |
| `getAssets().open()` 加载资源 | `public/` 目录或 CDN |
| 录音保存到 SD 卡 | `URL.createObjectURL` 生成 blob URL 预览/发送 |
| 竖屏手机布局 | 响应式布局（移动端优先） |
| Android 146 色 `R.color.white+i` | CSS 变量 `--color-0` 到 `--color-145` |
